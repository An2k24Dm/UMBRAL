# HU18 — Activar Trivia

**Como** Operador, **quiero** activar una trivia previamente creada y configurada, **para** que esté disponible para ser utilizada en futuras sesiones de juego.

---

## Criterios de aceptación

- Solo se puede activar una trivia en estado **Borrador** o **Archivada**.
- La trivia debe tener **al menos una pregunta** antes de activarse.
- Al activarse, se genera un evento de integración `TriviaActivada` en la tabla Outbox.
- El evento Outbox incluye: `TriviaId`, `Nombre`, `CantidadPreguntas`.

---

## Flujo

```
Frontend → PATCH /api/juegos/trivias/{triviaId}/activar
           Authorization: Bearer <token-operador>
        ↓
TriviasControlador.ActivarTrivia(triviaId)
        ↓
ActivarTriviaComando(triviaId, operadorId) → MediatR
        ↓
ActivarTriviaManejador:
  1. repositorio.ObtenerTriviaPorIdAsync(triviaId)
       → si no existe → ExcepcionNoEncontrado (404)
  2. trivia.Activar()
       → valida estado Borrador o Archivada  ← si Activa → ExcepcionDominio (422)
       → valida al menos 1 pregunta          ← si 0 → ExcepcionDominio (422)
       → Estado = Activa
       → registra TriviaActivadaEvento (in-memory)
  3. repositorio.ActivarTriviaAsync(trivia)
       → UPDATE Trivia SET estado = 1
       → INSERT EventoSalida (Outbox) en misma transacción
        ↓
HTTP 204 No Content
```

---

## Endpoint

| Método  | Ruta | Descripción |
|---------|------|-------------|
| `PATCH` | `/api/juegos/trivias/{triviaId}/activar` | Activar trivia |

---

## Outbox (Patrón de integración)

El evento se guarda en la tabla `juegos.EventoSalida` con `Procesado = false`.
Un publicador externo (futuro) lo leerá y enviará a **RabbitMQ** para que `partidas-servicio` sepa qué trivias están disponibles.

```json
{
  "tipo": "TriviaActivada",
  "datos": {
    "TriviaId": "...",
    "Nombre": "Trivia de Geografía",
    "CantidadPreguntas": 5
  }
}
```
