# HU19 — Modificar Trivia

**Como** Operador, **quiero** modificar los detalles de una trivia (nombre, descripción, tiempo límite), **para** corregir información antes de activarla o incluso estando activa.

---

## Criterios de aceptación

- Se puede modificar una trivia en estado **Borrador** o **Activa**.
- No se puede modificar una trivia **Archivada**.
- Si la trivia está **Activa**, se genera un evento de integración `TriviaModificada` en el Outbox para que otros servicios actualicen su caché.
- No se modifican las preguntas (eso es HU16/17).

---

## Flujo

```
Frontend → PUT /api/juegos/trivias/{triviaId}
           Authorization: Bearer <token-operador>
           { nuevoNombre, nuevaDescripcion, nuevoTiempoLimitePorPregunta }
        ↓
TriviasControlador.ModificarTrivia(triviaId, ModificarTriviaDto)
        ↓
ModificarTriviaComando(triviaId, dto) → MediatR
        ↓
ModificarTriviaManejador:
  1. repositorio.ObtenerTriviaPorIdAsync(triviaId)
       → si no existe → ExcepcionNoEncontrado (404)
  2. trivia.ModificarDatos(nuevoNombre, nuevaDescripcion, nuevoTiempo)
       → si Archivada → ExcepcionDominio (422)
       → valida nombre, descripción y tiempo
       → actualiza campos + registra TriviaModificadaEvento
  3. repositorio.ModificarDatosTriviaAsync(trivia)
       → UPDATE Trivia SET nombre, descripcion, tiempo_limite_por_pregunta
       → INSERT EventoSalida si el estado es Activa
        ↓
HTTP 204 No Content
```

---

## Endpoint

| Método | Ruta | Descripción |
|--------|------|-------------|
| `PUT`  | `/api/juegos/trivias/{triviaId}` | Modificar datos generales de una trivia |
