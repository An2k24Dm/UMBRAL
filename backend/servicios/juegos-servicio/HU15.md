# HU15 — Crear Trivia

**Como** Operador, **quiero** crear una nueva trivia estableciendo su nombre, descripción y parámetros generales, **para** luego agregarle preguntas antes de activarla.

---

## Criterios de aceptación

- El operador puede crear una trivia proporcionando nombre, descripción y tiempo límite por pregunta.
- La trivia se crea en estado **Borrador**.
- No pueden existir dos trivias con el mismo nombre.
- El operador puede ver la lista de sus trivias en borrador.
- El operador puede ver el detalle completo de una trivia (incluyendo preguntas y opciones).
- El `creadorId` se extrae del token JWT; el frontend no lo envía en el body.

---

## Flujo

```
Frontend → POST /api/juegos/trivias
           Authorization: Bearer <token-operador>
           { nombre, descripcion, tiempoLimitePorPregunta }
        ↓
TriviasControlador.CrearTrivia(CrearTriviaDto)
        ↓
CrearTriviaComando(dto, creadorId) → MediatR
        ↓
CrearTriviaManejador:
  1. ExisteTriviaConNombreAsync(nombre) → si existe → ExcepcionDominio (422)
  2. Trivia.Crear(nombre, descripcion, creadorId, tiempo, fechaHoraUtc)
       → Estado: Borrador
       → Registra TriviaCreadaEvento (in-memory)
  3. repositorio.AgregarTriviaAsync(trivia)
        ↓
HTTP 201 Created  { id: "guid-de-la-trivia" }
```

---

## Endpoints

| Método | Ruta | Descripción |
|--------|------|-------------|
| `POST` | `/api/juegos/trivias` | Crear trivia en borrador |
| `GET`  | `/api/juegos/trivias/borrador` | Listar trivias en borrador del operador autenticado |
| `GET`  | `/api/juegos/trivias/{id}` | Detalle completo de una trivia |
