# HU22 — Agregar Etapa a Búsqueda del Tesoro

**Como** Operador, **quiero** agregar etapas ordenadas a una búsqueda del tesoro en borrador para estructurar el recorrido de los participantes.

---

## Criterios de aceptación

- Solo se pueden agregar etapas a búsquedas en estado **Borrador** o **Activa** (no Archivada).
- El orden de la etapa se asigna automáticamente según la cantidad de etapas existentes.
- Si la búsqueda no existe, se retorna 404.
- Si la búsqueda está archivada, se retorna 422.

---

## Flujo — Agregar etapa

```
Frontend → POST /api/juegos/busquedas/{busquedaId}/etapas
           Authorization: Bearer <token-operador>
           Body: { titulo, descripcion }
        ↓
AgregarEtapaComando(busquedaId, dto) → MediatR
        ↓
AgregarEtapaManejador:
  1. repositorio.ObtenerBusquedaPorIdAsync(busquedaId)
       → si no existe → ExcepcionNoEncontrado (404)
  2. busqueda.AgregarEtapa(titulo, descripcion)
       → si Archivada → ExcepcionDominio (422)
       → orden = etapas.Count + 1
  3. repositorio.AgregarEtapaAsync(busquedaId, etapa)
       → INSERT Etapa
        ↓
HTTP 201 Created  { id: guid }
```

---
