# HU20 — Eliminar / Desactivar Trivia

**Como** Operador, **quiero** archivar una trivia para que no pueda ser elegida en futuras sesiones, conservando el historial de las partidas pasadas.

---

## Criterios de aceptación

- **Nunca** se elimina físicamente una trivia de la base de datos (no hay `DELETE` en la tabla).
- La trivia pasa a estado **Archivada** (soft delete / borrado lógico).
- No se puede archivar una trivia que ya esté **Archivada**.
- Al archivarse, se genera el evento de integración `TriviaArchivada` en el Outbox.
- El `partidas-servicio` (futuro) recibirá este evento y quitará la trivia de su catálogo de selección.
- Los Operadores pueden consultar únicamente trivias **Activas** para crear sesiones.

---

## Flujo — Desactivar

```
Frontend → DELETE /api/juegos/trivias/{triviaId}
           Authorization: Bearer <token-operador>
        ↓
DesactivarTriviaComando(triviaId, operadorId) → MediatR
        ↓
DesactivarTriviaManejador:
  1. repositorio.ObtenerTriviaPorIdAsync(triviaId)
       → si no existe → ExcepcionNoEncontrado (404)
  2. trivia.Desactivar()
       → si ya Archivada → ExcepcionDominio (422)
       → Estado = Archivada
       → registra TriviaArchivadaEvento
  3. repositorio.ArchivarTriviaAsync(trivia)
       → UPDATE Trivia SET estado = 2  ← Archivada (NO DELETE)
       → INSERT EventoSalida (Outbox)
        ↓
HTTP 204 No Content
```

## Flujo — Obtener trivias activas

```
Frontend → GET /api/juegos/trivias/activas
           Authorization: Bearer <token-operador>
        ↓
ObtenerTriviasActivasConsulta() → MediatR
        ↓
ObtenerTriviasActivasManejador:
  repositorio.ObtenerTriviasActivasAsync()
    → SELECT ... WHERE estado = 1 ORDER BY nombre
        ↓
HTTP 200 OK  [ TriviaResumenDto... ]
```

---

## Endpoints

| Método   | Ruta | Descripción |
|----------|------|-------------|
| `DELETE` | `/api/juegos/trivias/{triviaId}` | Archivar trivia (soft delete) |
| `GET`    | `/api/juegos/trivias/activas` | Listar trivias activas (para crear sesiones) |
