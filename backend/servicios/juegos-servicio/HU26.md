# HU26 — Activar Búsqueda del Tesoro

**Como** Operador, **quiero** activar una Búsqueda del Tesoro configurada **para** que esté disponible en el catálogo al crear sesiones de juego.

---

## Criterios de aceptación

- El estado actual debe ser **Borrador**.
- La búsqueda debe tener **al menos una etapa**.
- Cada etapa debe tener **al menos una misión** (si alguna etapa está vacía → 422).
- Al activar, el estado cambia a **Activa** y se emite un evento de dominio `BusquedaTesoroActivada` vía patrón Outbox.
- Las búsquedas activas aparecen en el listado `GET /activas`.

---

## Flujo — Activar

```
Frontend → PATCH /api/juegos/busquedas/{busquedaId}/activar
           Authorization: Bearer <token-operador>
        ↓
ActivarBusquedaTesoroComando(busquedaId, operadorId) → MediatR
        ↓
ActivarBusquedaTesoroManejador:
  1. repositorio.ObtenerBusquedaPorIdAsync(busquedaId)
       → si no existe → ExcepcionNoEncontrado (404)
  2. busqueda.Activar()
       → si no está en Borrador → ExcepcionDominio (422)
       → si no tiene etapas → ExcepcionDominio (422)
       → si alguna etapa no tiene misiones → ExcepcionDominio (422)
       → Estado = Activa
       → BusquedaActivadaEvento (in-memory)
  3. repositorio.ActivarBusquedaTesoroAsync(busqueda)
       → UPDATE estado = Activa
       → INSERT EventoSalida { Tipo = "BusquedaTesoroActivada" }
        ↓
HTTP 204 No Content
```

## Flujo — Listar activas

```
Frontend → GET /api/juegos/busquedas/activas
           Authorization: Bearer <token-operador>
        ↓
ObtenerBusquedasActivasConsulta → MediatR
        ↓
ObtenerBusquedasActivasManejador:
  1. repositorio.ObtenerBusquedasActivasAsync()
       → SELECT * FROM BusquedaTesoro WHERE estado = Activa ORDER BY nombre
        ↓
HTTP 200 OK  List<BusquedaTesoroResumenDto>
```

---

## Endpoints

| Método  | Ruta                                          | Descripción                              |
|---------|-----------------------------------------------|------------------------------------------|
| `PATCH` | `/api/juegos/busquedas/{busquedaId}/activar`  | Activar búsqueda (Borrador → Activa)     |
| `GET`   | `/api/juegos/busquedas/activas`               | Listar búsquedas activas                 |

---

## Integración asíncrona (Outbox)

Al activar se inserta un registro en `juegos.EventoSalida`:
```json
{
  "tipo": "BusquedaTesoroActivada",
  "datos": {
    "BusquedaId": "guid",
    "Nombre": "string",
    "CantidadEtapas": 3
  }
}
```
