# HU23 — Agregar Misión a Etapa

**Como** Operador, **quiero** agregar misiones a las etapas de una búsqueda del tesoro para definir las tareas que los participantes deberán completar en cada fase.

---

## Criterios de aceptación

- Solo se pueden agregar misiones a búsquedas que no estén **Archivadas**.
- La etapa debe existir dentro de la búsqueda indicada.
- Si la búsqueda o la etapa no existen, se retorna 404.
- Si la búsqueda está archivada, se retorna 422.
- El tipo de misión se envía como entero: `0=PistaTexto`, `1=Acertijo`, `2=CodigoQR`.

---

## Flujo — Agregar misión

```
Frontend → POST /api/juegos/busquedas/{busquedaId}/etapas/{etapaId}/misiones
           Authorization: Bearer <token-operador>
           Body: { titulo, descripcion, tipo, pistaClave }
        ↓
AgregarMisionComando(busquedaId, etapaId, dto) → MediatR
        ↓
AgregarMisionManejador:
  1. repositorio.ObtenerBusquedaPorIdAsync(busquedaId)
       → si no existe → ExcepcionNoEncontrado (404)
  2. busqueda.AgregarMisionAEtapa(etapaId, titulo, descripcion, tipo, pistaClave)
       → si Archivada → ExcepcionDominio (422)
       → si etapa no encontrada → ExcepcionNoEncontrado (404)
       → etapa.AgregarMision(...)
  3. repositorio.AgregarMisionAsync(etapaId, mision)
       → INSERT Mision
        ↓
HTTP 201 Created  { id: guid }
```

---

## Endpoints

| Método | Ruta | Descripción |
|--------|------|-------------|
| `POST` | `/api/juegos/busquedas/{busquedaId}/etapas/{etapaId}/misiones` | Agregar misión a una etapa |

---

## Tipos de misión (TipoMision)

| Valor | Nombre | Descripción |
|-------|--------|-------------|
| `0` | `PistaTexto` | Pista entregada como texto |
| `1` | `Acertijo` | Pregunta o acertijo que el participante debe resolver |
| `2` | `CodigoQR` | Código QR que el participante debe escanear |
